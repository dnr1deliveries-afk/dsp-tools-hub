Framework Compliance

DSP Tools Hub v2.0 — Compliant with On-Road DSP Collaboration SOP (Week 21)

What Changed in v2.0

All tools now output DSP-level summary data only. Route-level detail, DA/driver identifiers, and action requests have been removed to comply with the On-Road DSP Collaboration SOP.

DSP-Level Data Only

All metrics are aggregated to DSP totals. No route-level breakdown, no DA/TRID identifiers.

Informational Only

Messages provide awareness data only. No action requests, no "please do X" language.

Support-Driven

Data shared for DSP awareness and self-service. DSPs access detailed data via their own tools.

What Was Removed
❌ Route-Level Data
  • Individual route IDs
  • Per-route package counts
  • Route-level bag grouping
  • Route lookup features
❌ DA/Driver-Level Data
  • Transporter IDs (even anonymised)
  • DA performance by individual
  • VINs and VRNs
  • DA deployment patterns
❌ Action Requests
  • "Please chase"
  • "Provide root cause"
  • "Please provide insight"
  • "Confirm when completed"
❌ Directive Language
  • "You must..."
  • "You need to..."
  • "Action required"
  • Route/DA-level flags
Framework Reference

This tool complies with the On-Road DSP Collaboration SOP (Week 21), which establishes:

  • Four authorised meeting types: DDM, DORM, DPR, DSP Roundtables
  • DSP autonomy: DSPs are independent business owners who manage their own operations
  • Support-driven model: OPS provides resources when DSPs request them
  • DSP-level data only: No route-level or DA-level data sharing

For detailed framework information, refer to the On-Road DSP Collaboration SOP and Principles of Engagement (POE) guidelines.
